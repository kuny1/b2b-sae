# B2B 行业信息
## 商业洞察：行业理解 & 企业定位
- [通识]企业存在的本质：通过创造客户价值，高效地获取利润并持续生存下去
- B2B企业：用数字化的方式，重构一条跨越国界、充满摩擦的传统贸易链条
- 平台价值
  - 发现：降低“买卖双方互相找到”的成本（信息撮合）
  - 信任：降低“互相确认对方靠谱”的成本（信用背书）
  - 交易：降低“确认规格、谈价格、签合同”的成本（交易服务）
  - 履约：降低“交付、支付、通关、物流”的成本（履约服务）
- 盈利模式：用技术和规则，降低了B2B跨境交易中因“信息不对称、信任缺失、流程复杂”而产生的摩擦成本，在交易的4个阶段中部分或者全部提供价值，从中分成。
  - 早期，平台价值在于“信息撮合”——赚的是“发现费”（会员费/广告费）。
  - 中期，平台价值在于“交易保障”——赚的是“信任费”（佣金/担保费）。
  - 长期，平台价值在于“履约闭环”——赚的是“效率费”（物流/金融/数据服务）


### 行业理解
- 行业本质：是中国过剩的高效供应链与全球碎片化采购需求之间的“连接器”和“信任中介”
> 表层是“撮合”，“核心”是履约，底层是“信任”
- 核心要素：信任（起点）、效率（杠杆）、规模（终点）
- 行业痛点：信息不对称与信任成本高、交易流程复杂且低效、规模化边际成本高
- 交易链路：获客与匹配(资信审查)、规格确认与报价、生产与跟单、跨境物流与清关、支付与结算(售后)
- 分析框架：（现状、前景、护城河&竞争优势）
  - 天时：大环境怎么样？
  - 地利：战场选在哪？
  - 人和：凭什么我们能赢？
    - 根基（客户）：客户凭什么掏钱？我们帮工厂解决了“客、钱、顺、稳”里的哪几个？（这是第一性原理，不可动摇的基石）
    - 敌情（对手）：同行在哪个点上强？在哪个点上弱？我们打哪里他能疼？
    - 我情（自己）：我们的技术、团队、供应链资源，能不能在那个“赢的点”上形成压倒性优势？

### 行业痛点
#### 信息不对称
| 问题                         | 详情                                                                                                |
| ---------------------------- | --------------------------------------------------------------------------------------------------- |
| 产品与工艺信息无法标准化     | 同品类不同工厂工艺、材料、品控差异巨大，无法用统一参数描述清楚                                      |
| 工厂产能与履约能力不透明     | 规模、排期、良品率、准时交付率等动态信息无法在交易前验证                                            |
| 买家真实需求与信誉不透明     | 采购预算、决策周期、真实采购量、付款历史——卖家对买家几乎一无所知                                    |
| 价格形成机制不透明           | 同一产品不同买家拿到的报价可能差30%，取决于采购量、关系、谈判能力、信息差——没有一个"公允价格"可参考 |
| 交易对手基本身份信息不可验证 | 工商注册、法人身份、经营资质、诉讼记录——跨境场景下这些基础信息验证成本极高                          |
#### 信任成本极高
| 问题               | 详情                                                                                                     |
| ------------------ | -------------------------------------------------------------------------------------------------------- |
| 法律威慑实质上失效 | 不是没有法律，而是跨法域诉讼成本远高于货值，对绝大多数中小订单来说，法律工具存在但用不起，威慑力归零     |
| 声誉传导机制断裂   | 信誉没法跨市场、跨交易积累和传递，每一笔新交易都是从零开始。违约不影响下次生意，因为根本没有'下次'的预期 |
| 物质保障不可执行   | 对方没有任何在平台或法律体系内可扣押的资产或资金，违约没有即时的物质代价                                 |
#### 交易流程复杂（结构性根源）
> 非标定制与跨国多节点是客观存在的复杂结构，直接催生了后续的低效表现。

**前段是“非标定制”而非“标准品买卖”**
- 每笔订单的规格、工艺、交期、付款条件都不同，不存在通用模板
- 天然依赖大量反复沟通才能完成确认，结构上就无法标准化

**后段是跨国的物理交付，涉及硬性节点与角色切换**
- 必须经历装柜、报关、海运、清关、尾程配送等多个节点
- 每个节点有不同角色介入，责任方、系统、信息不断切换，复杂度呈指数级放大

#### 交易流程低效（直接表现）
> 由上述结构性复杂直接导致的执行层面问题，使每一次交易都消耗极高的时间和人力成本。

**信息确认是“往返式”的**
- 规格、价格、交期、付款每一项都需要来回确认
- 跨时区下一个回合就是24小时，单个订单几十个回合很常见
- 瓶颈不在人动作慢，而在于“确认一件事需要多少次来回”的结构锁定

**信息流转是“碎片化”的**
- 询盘在平台，报价在邮件，规格在微信，物流在货代系统，付款在银行
- 信息散落在互不联通的载体中，完全靠人工搬运（复制粘贴、转发、导入导出）
- 搬运过程中频繁出现格式转换错误和数据丢失，一致性与连贯性无法保障

#### 难以规模化（人力依赖的必然结果）
> 市场经济的本质是资源的流动。任何阻碍资源流动的系统终将被改进或替代。当前模式下，核心环节高度依赖人，机器难以介入。

**决策靠人**
- 产品非标、需求不透明、价格机制缺失——信息不对称未解决
- 机器无法基于残缺数据做出判断，只能依靠人的经验和直觉来弥补信息缺口

**交付靠人**
- 流程多节点、多角色，信息跨系统碎片化，沟通多方异步——流程复杂与低效并存
- 在这些问题解决之前，机器无法自动串联执行，只能靠人工催办、协调、跟进
